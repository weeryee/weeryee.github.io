    知识图谱概览

    weeryee 9/18/2022 知识图谱KG

    # 一.语言与知识

    # 1.构建有学识的人工智能

    早期人工智能两大流派 连接主义：感知，识别，判断——》深度学习（聪明的AI） 符号主义：语言，认知，知识（有学识的AI）——》知识图谱（有学识的AI）

    认知智能的两个核心命题：

    1. 语言理解
    2. 知识的表示与处理

    # 2.知识的承载与表示方式

    什么是知识： 人类的自然语言，以及创作的绘画和音乐、数学语言、物理模型、化学公式等都是人类知识的表示形式和传承方式。具有获取、表示和处理知识的能力是人类心智区别于其它物种心智的最本质特征，也是人脑智能的最本质特征。 人类通过认识世界积累知识，并通过语言来描述、记录和传承关于世界的知识。同时，准确理解语言也极大依赖大脑中所习得的各种知识

    知识表示方式： 知识有很多种表达载体和存在形式，如自然语言，知识图谱以及向量表示。

    • 自然语言(Text as Knowledge Base)是人类知识的最主要表达载体，但对于机器不太友好。通过机器理解文本有两大主要路线：
      • 抽取技术，通过从文本中提取实体，关系，逻辑结构等
      • 语言预训练，即通过大量文本语料训练一个神经网络大模型，文本中的知识被隐含在参数化的向量模型中。
    • 知识图谱(KG as Knowledge Base)的本质是一种知识的表示形式。简单来说，知识图谱值在利用图结构建模，识别和推断事务之间的复杂关联关系和沉淀领域知识。相比文本而言，结构化数据更易于被机器处理。同时图结构比起字符串序列能表达更丰富的语义和知识。
    • 参数化的向量和神经网络(Embeddings as Knowledge Base)是更适合机器完成快速计算的信息载体。我们通常把这些向量化表示称为Embedding或“Distributed Vector Representation”。如果将词语，实体，对象和关系等都投影到向量空间，就可以更方便的在向量空间对这些语言，视觉和实体对象进行操作，甚至可以利用神经网络实现逻辑推理。

    # 二.知识图谱的起源

    # 知识图谱的互联网基因

    Vannevar Bush提出一个MEMEX的记忆机器，即通过mesh关联网络来存储电子化的百科全书。该设想启发了Hypertext的实现，而超文本技术促成了World Wide Web的发明。

    "Wholly new forms of encyclopedias will appear, ready made witha mesh of associative trails running through them, ready to be dropped into the memex and there amplified”

    # 数据的互联网（Semantic Web）

    Semantic Web（语义网）与经典web一样，是以图和链接为中心的信息管理系统，但其图中节点表示粒度更细的事物，如一本书，个人，机构等，同时图中的链接也标明了事物之间的语义关系。

    # 知识图谱的概念演进

    知识图谱并非突然出现的全新技术，而是很多相关领域不断发展融合的结果。一方面，知识图谱具有人工智能的基因。而另一方面，知识图谱也具有鲜明的互联网基因，如互联网促进人类知识的共享和开放。

    # 三.知识图谱的价值

    • 语义搜索
    • 智能问答
    • 推荐系统
    • 辅助语言语义理解
    • 拓展视觉理解的深度和广度
    • 辅助IoT设备互联
    • 大数据分析

    # 四.知识图谱的技术内涵

    # 知识图谱是交叉技术领域

    1. 在人工智能和机器学习领域，传统符号知识表示是知识图谱的重要基础技术；
    2. 在深度学习、表示学习等领域与知识图谱的交叉产生了知识图谱嵌入、知识图谱表示学习等交叉领域。
    3. 在传统的自然语言处理领域，怎样从文本中自动化识别实体、抽取关系、检测事件等信息一直是经久不衰的研究热题，Knowledge Base Population即是知识图谱与自然语言处理之间产生的交叉领域。
    4. 在数据库领域，知识图谱与数据库的交叉又产生了图数据库，而图嵌入、图挖掘等数据挖掘领域的方法也广泛地被用来处理知识图谱数据。

    # 知识图谱的两个核心技术维度

    1. 第一个是从知识的视角，它来源于传统AI的知识表示与推理领域，关心怎么表示概念和实体，怎样刻画它们之间的关系，怎样进一步表示公理、规则等更加复杂的知识。
    2. 第二个视角是从图的视角，它来源于知识图谱的互联网基因，关心图中的节点、边、链接、路径、子图结构，怎样存储大规模的图数据，怎样利用图的结构对图数据进行推理、挖掘与分析等。

    # 知识图谱的技术栈

    # 1.基于图的知识表示

    知识图谱就是基于有向标记图的知识表示方法，常用的表示法有属性图和RDF图两种。

    1. 属性图

      • 节点(Nodes)：是图中的实体，用表示其类型的0到多个文本标签进行标记，相当于实体。
      • 边(Edges)：是节点之间的定向链接，也称为关系。其中对应的“from node”称为源节点，“to node”称为目标节点。边是定向的且每条边都有一个类型，它们可以在任何方向上导航和查询。相当于实体之间的关系。
      • 属性(Properties)：是一个键值对，顶点和边都具有属性。

    2. RDF图 RDF（Resource Description Framework）使用标准的图数据模型，其技术栈的标准是由万维网联盟(W3C)管理的。RDF有一个标准的查询语言称为SPARQL。

      RDF图数据模型主要是由以下两个部分组成的：

      • 节点(Nodes)：对应图中的顶点，可以是具有唯一标识符的资源，也可以是字符串、整数等有值的内容。
      • 边(Edges)：是节点之间的定向链接，也称为谓词或属性。边的入节点称为主语，出节点称为宾语，由一条边连接的两个节点形成一个主语-谓词-宾语的陈述，也称为三元组。边是定向的，它们可以在任何方向上导航和查询。

    # 2.图数据存储与查询

    图数据库充分利用图的结构建立微索引。这种微索引比关系数据库的全局索引在处理图遍历查询时更加廉价，其查询复杂度与数据集整体大小无关，仅正比于相邻子图的大小。
